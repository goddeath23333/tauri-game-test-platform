# 服务器数据库端部署文档

## 1. 概述

本文档面向运维人员，提供 Windows Server 环境下的一键部署方案。

### 1.1 目标环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows Server 2016+ |
| 数据库 | PostgreSQL 15+ / MySQL 8+ |
| 内存 | 最低 4GB，推荐 8GB |
| 磁盘 | 最低 50GB |

### 1.2 部署特点

- **一键部署**：开箱即连，无需手动配置
- **可视化运维**：TUI/GUI 工具，运维人员无需懂代码
- **最高权限**：完整的数据库增删改查能力
- **端口管理**：部署端口与出口端口独立配置

---

## 2. 一键部署包

### 2.1 部署包内容

```
gametester-server-deploy/
├── install.bat              # 一键安装脚本
├── start.bat                # 启动服务
├── stop.bat                 # 停止服务
├── config.ini               # 配置文件
├── admin-tool.exe           # 运维管理工具 (TUI)
├── admin-tool-gui.exe       # 运维管理工具 (GUI)
├── database/
│   ├── postgres-portable/   # PostgreSQL 便携版
│   └── init.sql             # 数据库初始化脚本
├── server/
│   └── gametester-server.exe
└── logs/                    # 日志目录
```

### 2.2 一键安装

**步骤 1：下载部署包**

从 GitHub Releases 下载最新部署包：
```
https://github.com/gametester/server/releases
```

**步骤 2：解压到目标目录**

```
解压到: C:\GameTester\Server\
```

**步骤 3：运行一键安装**

双击 `install.bat` 或在命令行执行：

```batch
cd C:\GameTester\Server
install.bat
```

安装脚本会自动完成：
- 检测系统环境
- 安装 PostgreSQL 便携版
- 初始化数据库
- 配置端口
- 创建管理员账户
- 启动服务

## 3. 端口配置

### 3.1 端口说明

| 端口类型 | 默认值 | 说明 |
|---------|--------|------|
| 部署端口 | 8080 | API 服务监听端口 |
| 出口端口 | 8080 | 外部访问端口 |
| 数据库端口 | 5432 | PostgreSQL 内部端口 |

### 3.2 配置文件 (config.ini)

```ini
[server]
; API服务部署端口（内部监听）
deploy_port=8080

; 出口端口（外部访问，用于防火墙配置）
external_port=8080

; 绑定地址
host=0.0.0.0

[database]
; 数据库类型: postgres, mysql
type=postgres

; 数据库端口
port=5432

; 数据库名称
name=gametester

; 最高权限账户
admin_user=gametester_admin
admin_password=请修改此密码

[security]
; 是否启用SSL
ssl_enabled=false

; SSL证书路径
ssl_cert=
ssl_key=

[logs]
; 日志级别: debug, info, warn, error
level=info

; 日志保留天数
retention_days=30
```

### 3.3 修改端口

**方式一：编辑配置文件**

修改 `config.ini` 中的端口值，然后重启服务：

```batch
stop.bat
start.bat
```

**方式二：使用运维工具**

运行 `admin-tool.exe`，选择「端口配置」菜单。

## 10. 防火墙配置

### 10.1 Windows防火墙规则

**自动配置（推荐）：**

安装脚本会自动添加防火墙规则。

**手动配置：**

```powershell
# 添加入站规则（API端口）
netsh advfirewall firewall add rule name="GameTester API" dir=in action=allow protocol=tcp localport=8080

# 添加入站规则（数据库端口，仅内部）
netsh advfirewall firewall add rule name="GameTester DB" dir=in action=allow protocol=tcp localport=5432 remoteip=127.0.0.1

# 添加出站规则
netsh advfirewall firewall add rule name="GameTester Out" dir=out action=allow protocol=tcp localport=8080
```

### 10.2 端口开放检查

```
选择 [6] 系统诊断 → [2] 端口检测

查看防火墙规则是否正确配置。
```

## 11. 常见问题

### 11.1 端口被占用

```
问题: 启动失败，提示端口8080被占用

解决:
1. 运行 admin-tool.exe
2. 选择 [4] 端口配置 → [1] 修改部署端口
3. 输入新端口: 8081
4. 重启服务
```

### 11.2 数据库连接失败

```
问题: 数据库连接失败

解决:
1. 运行 admin-tool.exe
2. 选择 [6] 系统诊断 → [3] 连接测试
3. 查看具体错误信息
4. 如果数据库未启动，选择 [1] 服务管理 → [3] 重启服务
```

### 11.3 服务无法启动

```
问题: 服务启动失败

解决:
1. 查看日志: [5] 日志查看 → [1] 实时日志
2. 运行诊断: [6] 系统诊断 → [1] 环境检测
3. 检查配置文件 config.ini 是否正确
```

### 11.4 外部无法访问

```
问题: 外部客户端无法连接服务器

解决:
1. 检查防火墙: [6] 系统诊断 → [2] 端口检测
2. 确认出口端口配置正确
3. 确认 config.ini 中 host=0.0.0.0
```
