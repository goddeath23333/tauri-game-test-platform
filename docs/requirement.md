# 软件需求文档

## 1. 项目概述

**项目名称**：游戏测试平台
**项目类型**：Tauri 桌面应用
**核心功能**：游戏测试Issue管理与工作流程跟踪

---

## 2. 用户角色

| 角色 | 职责 |
|------|------|
| 测试人员 | 测试游戏、提交Issue、验证修复 |
| 开发人员 | 回复工单、修复Issue |

---

## 3. 功能需求

### 3.1 Issue管理
- 创建Issue（标题、描述、严重程度、截图）
- 查看Issue列表（支持状态筛选）
- 查看Issue详情（包含评论历史）
- 修改Issue状态

### 3.2 工作流程
```
测试人员测试 → 提交Issue → 人员工单回复 → 开始修复 → 修复完成 → 测试人员确认修复完成
```

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| Pending | 待处理 | 开发人员可开始修复 |
| InProgress | 处理中 | 开发人员可完成修复 |
| Fixed | 已修复 | 测试人员可确认修复 |
| Verified | 已验证 | 流程结束 |

### 3.3 评论系统
- 开发人员回复工单
- 测试人员评论
- 记录完整对话历史

---

## 4. 非功能需求

- 本地数据存储（SQLite）
- Windows桌面应用
- 快速启动（<3秒）
