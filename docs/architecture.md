## 2. 部署模式

### 2.1 独立模式（本地优先）
- 仅使用本地 SQLite 数据库
- 无需服务器，适合个人使用
- 数据存储在本地

### 2.2 联网模式（服务器同步）
- 本地 SQLite + 远程 PostgreSQL/MySQL
- 数据可与团队共享
- 支持多用户协作

## 3. 数据存储策略

### 3.1 本地轻量化存储

本地SQLite仅存储必要数据，保持轻量：

| 本地数据 | 说明 | 保留策略 |
|---------|------|----------|
| 用户偏好 | 主题、语言、同步设置 | 永久保留 |
| 最近Issue缓存 | 最近50条Issue摘要 | 定期清理 |
| 离线操作队列 | 未同步的操作记录 | 同步后删除 |
| 标签列表 | 全部标签（轻量） | 同步更新 |
| 用户列表 | 全部用户摘要（姓名+岗位） | 同步更新 |

### 3.2 服务器核心数据存储

服务器存储完整业务数据：

| 服务器数据 | 说明 | 存储方式 |
|-----------|------|----------|
| Issue完整数据 | 标题、描述、截图、历史 | PostgreSQL/MySQL |
| 评论历史 | 全部评论记录 | PostgreSQL/MySQL |
| 用户完整信息 | 认证、统计、权限 | PostgreSQL/MySQL |
| 状态变更历史 | 全部状态流转记录 | PostgreSQL/MySQL |
| 附件文件 | 截图、日志文件 | 文件服务器 |

### 3.3 联网同步策略

```
同步触发条件:
- 应用启动时自动同步
- 定时同步（可配置间隔）
- 手动点击"同步"按钮
- Issue状态变更时立即同步

同步流程:
┌─────────────────────────────────────────────────────────────┐
│                      数据同步流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 检查网络连接                                             │
│     └───────────────────────────────────────┐              │
│                                             │              │
│  2. 上传离线操作队列                          │              │
│     - 创建的Issue                            │              │
│     - 状态变更                                │              │
│     - 新增评论                                │              │
│                                             │              │
│  3. 下载服务器更新                           │              │
│     - 新Issue                                │              │
│     - 状态变更                                │              │
│     - 新评论                                  │              │
│                                             │              │
│  4. 合并本地缓存                             │              │
│     - 更新最近Issue列表                       │              │
│     - 更新用户/标签列表                       │              │
│                                             │              │
│  5. 清理已同步的离线队列                      │              │
│                                             │              │
│  6. 更新同步时间戳                           │              │
│                                             │              │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 离线模式

```
离线时:
- 本地创建Issue → 加入离线队列
- 本地修改状态 → 加入离线队列
- 本地添加评论 → 加入离线队列
- 查看缓存数据 → 正常显示

联网后:
- 自动上传离线队列
- 下载最新数据
- 合并冲突（服务器优先）
```

## 4. 请求人/负责人逻辑

### 4.1 角色定义

```
请求人 (requester): 提交Issue的人
负责人 (assignee): 被指派处理Issue的人

典型场景:
- 测试人员(test) → 请求人
- 开发人员(developer) → 负责人
- 管理员(admin) → 可指派负责人
```

### 4.2 指派流程

```
┌─────────────────────────────────────────────────────────────┐
│                      Issue指派流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 测试人员创建Issue                                        │
│     └───────────────────────────────────────┐              │
│                                             │              │
│  2. Issue状态: Pending                       │              │
│     requester: 测试人员                      │              │
│     assignee: 未指派                         │              │
│                                             │              │
│  3. 管理员/请求人指派负责人                   │              │
│     ────────────────────────────────┐      │              │
│                                      │      │              │
│  4. Issue状态: Pending               │      │              │
│     requester: 测试人员              │      │              │
│     assignee: 开发人员               │      │              │
│                                      │      │              │
│  5. 负责人点击"开始修复"              │      │              │
│     ────────────────────────────────│──────│              │
│                                      │      │              │
│  6. Issue状态: InProgress            │      │              │
│                                      │      │              │
│  7. 负责人可转交给其他开发人员        │      │              │
│     ────────────────────────────────│──────│              │
│                                      │      │              │
│  8. 负责人点击"标记已修复"            │      │              │
│     ────────────────────────────────│──────│              │
│                                      │      │              │
│  9. Issue状态: Fixed                 │      │              │
│                                      │      │              │
│  10. 测试人员点击"确认完成"           │      │              │
│     ────────────────────────────────│──────│              │
│                                      │      │              │
│  11. Issue状态: Verified             │      │              │
│                                      │      │              │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 指派规则

| 操作 | 执行者 | 条件 |
|------|--------|------|
| 创建Issue | tester/developer/admin | 无限制 |
| 指派负责人 | admin/requester | Issue状态为Pending |
| 转交负责人 | admin/assignee | Issue状态为InProgress |
| 开始修复 | admin/assignee | Issue状态为Pending |
| 标记已修复 | admin/assignee | Issue状态为InProgress |
| 确认完成 | admin/requester | Issue状态为Fixed |
| 修复未通过 | admin/requester | Issue状态为Fixed |

## 5. 权限控制

### 5.1 角色权限矩阵

| 功能 | admin | developer | tester |
|------|-------|-----------|--------|
| 创建Issue | ✅ | ✅ | ✅ |
| 查看Issue | ✅ | ✅ | ✅ |
| 修改Issue信息 | ✅ | ✅ | 仅自己创建的 |
| 指派负责人 | ✅ | ❌ | 仅自己创建的 |
| 开始修复 | ✅ | ✅(被指派) | ❌ |
| 标记已修复 | ✅ | ✅(被指派) | ❌ |
| 确认完成 | ✅ | ❌ | ✅(请求人) |
| 删除Issue | ✅ | ❌ | ❌ |
| 管理标签 | ✅ | ❌ | ❌ |
| 管理用户 | ✅ | ❌ | ❌ |
| 查看统计 | ✅ | ✅ | ✅ |

### 5.2 权限验证流程

```
用户请求 → JWT验证 → 角色检查 → 资源权限检查 → 执行操作
                ↓           ↓              ↓
              401错误    403错误        403错误
```

## 8. 数据流

### 本地操作
```
用户操作 → UI → Store → Tauri Command → Local SQLite → 返回
```

### 远程同步
```
本地数据变更 → 同步服务 → REST API → 服务器数据库 → 确认
```

### 权限验证
```
操作请求 → 权限检查器 → 角色验证 → 资源所有权验证 → 允许/拒绝
```

## 9. 核心模块

### Issue模块
- 创建Issue（本地优先）
- 查询Issue列表（本地缓存）
- 更新Issue状态（同步服务器）
- 获取Issue详情
- 指派/转交负责人

### 同步模块
- 自动同步开关
- 手动同步触发
- 冲突解决策略
- 离线队列

### 状态流转
```
Pending → InProgress → Fixed → Verified
         ↑____________↓ (修复未通过)
```
